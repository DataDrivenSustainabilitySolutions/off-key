# Contributing Guidelines

Thank you for contributing to this project! To maintain high code quality and ensure consistency, please follow the guidelines below.

---

## 🚀 Getting Started

1. Fork the repository.
2. Clone your fork locally:
   `git clone https://github.com/your-username/your-repo.git`
3. Create a feature branch from `main`:
   `git checkout -b your-feature-name`

---

## ✅ Pull Request Workflow

All changes must go through a pull request (PR). Direct pushes to `main` are blocked.

### PR Requirements:

- 📌 **1 Approval Required**: At least one team member must approve your PR before it can be merged.
- ✅ **CI/CD Checks Must Pass**: All required status checks (e.g., build, lint, tests) must complete successfully.
- 🔁 **New Commits Invalidate Approvals** *(if enabled)*: If you push additional commits after your PR is approved, the approval may be dismissed and require re-approval.

### To Create a Pull Request:

1. Push your feature branch:
   `git push origin your-feature-name`
2. Open a pull request to merge into `main`.
3. Request a review from at least one team member.
4. Wait for CI checks to pass and approvals to be received.
5. Once all requirements are met, you or a reviewer can merge the PR.

---

## 📎 Coding Standards

- Follow the existing code style and formatting.
- Write clear, concise commit messages.
- Include tests for new features or bug fixes when applicable.

---

## 🚫 Direct Pushes to `main`

> 🔒 **Direct pushes to `main` are not allowed.**
> All changes must be introduced through pull requests and approved by at least one reviewer.

---

## 📣 Communication

- If you're unsure about something, open a draft PR or start a discussion.
- Notify the team on Slack/email when your PR is ready for review.

---

Thank you for your contribution! 💙

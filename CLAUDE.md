# Full-Stack Development Guidelines
- Write pythonic, clean, well-structured, maintainable, and efficient Python code
- Follow Python coding conventions and best practices
- Build enterprise-grade solutions without quick fixes, workarounds, or anti-patterns
- Balance quality with simplicity - avoid over-engineering for a solo-developer mobile app
- Continuously enhance codebase quality and eliminate technical debt
- Seek clarification when requirements are unclear - never implement without full understanding
- Validate necessity and purpose before implementing any feature or change
- For debugging: analyze 5-7 potential root causes, narrow to 1-2 most likely, add logging to validate assumptions before implementing fixes
